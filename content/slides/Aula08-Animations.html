Animations

Na última aula vimos como criar uma transição de um estilo para outro.

Uma transição representa uma mudança de um estilo inicial para um estilo final, alterando suavemente os valores da propriedade especificada.

Porém uma transição só nos permite animar dessa forma, não podemos ter um estado intermediário.

Para esses casos, onde queremos criar animações com mais estados, fazemos o uso da propriedade animation

keyframes

@keyframes

É responsável por especificar como a animação deve funcionar. Para isso deve informar quais momentos ela deve alterar e qual estilo deve ser aplicado naquele instante

Exemplos

Propriedades

animation-name

Especifica o nome do keyframe que você deseja ligar ao seletor.

animation-duration

Especifica o tempo que uma animação deve durar. Se deixado vazio o valor default é 0s (sem animação).

animation-timing-function

Especifica a curva de velocidade da animação

animation-delay

Especifica um tempo para que a animação comece.

animation-iteration-count

Especifica quantas vezes uma animação deve repetir.

animation-direction

Especifica se a animação deve tocar em ciclos ou em modo reverso.

animation-fill-mode

Especifica qual estilo aplicar ao elemento quando a animação não estiver funcionando.

animation-play-state

Especifica se a animação está tocando ou não.

animation: shorthand-property

Aplica várias propriedades de uma só vez
